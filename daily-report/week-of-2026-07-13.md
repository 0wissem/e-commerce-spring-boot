# Weekly Report — Week of Monday 2026-07-13

> Engineering log for the e-commerce platform (Strangler Fig migration: monolith → microservices on AWS).
> One file per week. Each day: what I did, what I'm planning, and any problems and how I resolved them.

## Monday 2026-07-13

### What I did
No code changes committed today.

### What I'm planning
- Begin scaffolding the order-service Maven project: base directory structure, `pom.xml` with Spring Boot, Spring Data JPA, Flyway, PostgreSQL driver, and Spring Web — mirroring the product-service setup.
- Write the V1 Flyway migration for order-service: `orders` and `order_items` tables replicating the monolith schema so the new service can be shadow-validated before any traffic is cut over.
- Draft the hexagonal domain layer: `Order` and `OrderItem` entities plus the `IOrderRepository` port interface — no Spring or JPA imports, consistent with the pattern used across all other services.
- Add `/api/orders/**` to the gateway routing config with `ORDER_SERVICE_WEIGHT=0` as the initial env var, keeping all order traffic on the monolith until the new service passes end-to-end validation.

### Problems & solutions
None.
